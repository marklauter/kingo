A stored set-membership assertion — the triple resource#relationship@member, read set-first: the subject set contains the member, a subject, another subject set, or a resource (the thing itself, written `folder:y#...` — `...` is the production's marker, not a relationship). The member's shape belongs to the fact. The data-side aggregate root; created and deleted, never mutated.

## Contrasts

- `putative-fact` — the same shape held as a hypothesis rather than stored; the question Contains answers.
